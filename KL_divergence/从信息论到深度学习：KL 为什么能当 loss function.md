# 什么是熵（Entropy）

想象你在猜一个变量。如果它几乎确定（比如 99% 是 0)，你不需要太多信息。
如果它非常随机（10% 是0），你需要更多信息。
熵衡量的就是：

$$H(p)=−∑p(x)\log p(x)$$

**熵 = 平均不确定性 = 描述数据需要的最少信息量。**

# KL 散度是什么？

现在有两个分布：真实分布 p 和模型分布 q。

如果你用 q 去编码 p，平均要多花多少"比特"？

这就是：

$$KL(p∣∣q)=∑p(x)\log\frac{p(x)}{q(x)}$$

也可以写成：

$$KL(p∣∣q)=H(p,q)−H(p)$$

其中 $H(p,q)$ 是交叉熵。

**KL = 用错分布编码时，多浪费的信息量。**

# 三、为什么它能当训练目标？

关键关系式：

$$H(p,q) = H(p) + KL(p||q)$$

训练时：

- p 是真实分布（固定）
- H(p) 与模型无关

所以：

最小化 KL ⇔ 最小化交叉熵

✅**一句话结论：训练最小化交叉熵，本质是在逼近真实分布。**

---

# 四、分类任务里发生了什么？

分类任务里：

- 标签是 one-hot
- p 只有一个位置为 1

交叉熵变成：

$$-\log q(y_{true})$$

这就是：**Negative Log-Likelihood（NLL）**

softmax + CE 做的事：

1. softmax 产生概率
2. CE 计算 -log 正确类概率

所以：

最小化 CE ⇔ 最大化正确类概率 ⇔ 最大化似然

---

# 五、MLE 到底在干嘛？

最大似然估计：

$$\theta^* = \arg\max_\theta \prod q_\theta(y_i|x_i)$$

取 log：

$$\sum \log q_\theta(y_i|x_i)$$

最大化 log-likelihood ⇔ 最小化 NLL ⇔ 最小化交叉熵 ⇔ 最小化 KL(经验分布 || 模型分布)

✅**一句话结论：监督学习 = 经验分布向模型分布做 KL 投影。**

## 正文

### A. 直觉：KL(p‖q) 是"用 q 去编码 p 的额外代价/差距"

把 **p** 想成"真实世界会发生什么"，把 **q** 想成"你以为会发生什么"。如果你按 q 的概率去设计编码/预测，平均会多付出一笔"信息税"。

KL 的经典解释就是：当数据来自 p，却用为 q 优化的编码方案时，平均会多用多少比特（extra bits）。这就是"差距"的直觉来源。

训练时我们拿到的是样本（近似来自 p），模型给出 $q_\theta$。让 KL 变小，就是逼 $q_\theta$ 更像 p：对真实样本别再"惊讶"，给它更高概率。

✅**一句话结论：** KL 越小，说明模型的"概率表"越接近真实世界，因此它天然能当训练目标。

### B. Loss function 视角：最小化 KL 等价于最小化交叉熵（当 p 固定时）

关键关系式（先记住这一个就够用）：

$$H(p,q) = H(p) + KL(p||q)$$

当 p 固定（"真实分布/标签分布不变"），$H(p)$ 与模型无关，是常数。此时：

$$\min H(p,q) \Leftrightarrow \min KL(p||q)$$

密度估计里更直观：最小化 $KL(P_{data}||P_\theta)$ 等价于最大化期望 log-likelihood，因为"只差一个不含 $\theta$ 的项"。

✅**一句话结论：** 很多"交叉熵 loss"，本质是 KL 的"去常数版"；你在优化的是模型分布 $q_\theta$。

### C. 分类任务落地：one-hot 下，交叉熵为什么等价于 NLL；softmax + CE 怎么对应 MLE

多分类里常见流程是：网络吐出 logits（未归一化分数），再变成每类概率；交叉熵会惩罚"正确类概率太小"。

当标签是 one-hot，交叉熵只会留下"正确类那一项"，所以它就变成"正确类概率的负对数"——这就是 NLL 的样子。

所以 Softmax + CE 的训练含义非常朴素：让模型对正确类别的概率变大；从统计角度就是在做最大似然（MLE）。

工程细节常被新手写错：PyTorch 的 `CrossEntropyLoss` 直接吃 logits，等价于 `LogSoftmax + NLLLoss`（不用你手动 softmax）。

✅**一句话结论：** 在 one-hot 分类里，CE ≈ NLL；最小化 CE 就是在最大化"正确类的似然"。

### D. 与似然函数（log-likelihood / MLE）之间的关系

最大化似然（likelihood）与最大化对数似然（log-likelihood）等价，因为取对数不会改变最优点；把它取负，就得到 NLL（训练里要最小化的量）。

为什么会出现"平均 NLL/平均 CE"？因为 i.i.d. 假设下，全数据似然是连乘；取 log 变连加；再除以样本数得到"平均损失"。

把训练集看成经验分布 $\hat{p}$，把模型看成 $q_\theta$：经验交叉熵 $H(\hat{p},q_\theta)$ 就是经验 NLL；而 $KL(\hat{p}||q_\theta)$ 只比它少一个常数 $H(\hat{p})$。

从"分布拟合"角度看：MLE 等价于让模型分布逼近真实数据分布，一种常见表述就是在最小化 $KL(P_{data}||P_\theta)$。

✅**一句话结论：** MLE ⇔ max log-likelihood ⇔ min NLL；而 NLL 与 CE、KL 的差别主要是"常数项"和"p 的选取"。

### E. 常见误区澄清

- **误区：** KL 是对称"距离"。  
  **事实：** KL 不对称，也不满足严格距离（metric）的条件；它更像"信息差/代价差"。

- **误区：** 训练里随便用 $KL(p||q)$ 或 $KL(q||p)$ 都一样。  
  **事实：** 两者优化行为不同；文献里常说 forward KL 更"mean-seeking"，reverse KL 更"mode-seeking"。

- **误区：** 为什么训练常用 forward KL？  
  **关键原因：** MLE 对应的就是 $KL(P_{data}||P_\theta)$，因为期望在 $P_{data}$ 上，可用样本平均近似；而 $P_{data}$ 本身不可显式写出。

- **误区：** "CE = KL"永远成立。  
  **事实：** 严格关系是 $H(p,q)=H(p)+KL(p||q)$；只有在 $H(p)$ 为常数且被忽略（或 $H(p)=0$ 的退化情形）时，才可把它们当作同一个优化目标。

- **误区：** label smoothing 只是"技巧"，不改变 CE/KL 关系。  
  **事实：** 它直接把标签分布 p 从 one-hot 变成"正确类不再是 1、其余类分到一点概率"的分布，等价于改变了你要对齐的目标分布。

- **误区：** CE 是 loss，KL 是"距离度量"，所以一个能训练一个不能。  
  **事实：** 两者都能当目标；差别主要在定义语境——CE 更像"平均编码长度/平均惊讶"，KL 更像"相对多出来的编码长度"。

### F. 三个应用场景小结

**监督分类：** 最常见就是 Softmax + CE（one-hot 时等价 NLL），实现上常直接用 `CrossEntropyLoss` 吃 logits。

**知识蒸馏：** 经典做法是用 teacher 的"soft targets"训练 student，并在目标函数里用"对 soft targets 的交叉熵"；当 teacher 分布固定时，这等价于最小化 $KL(teacher||student)$（差一个常数）。

**VAE / 变分推断：** ELBO 里出现的 KL 项，可以直觉理解为"让编码器产生的潜变量分布别太偏离先验"，从而既能重构，又能让潜空间可采样。

✅**一句话结论：** 看到 loss 里出现 KL，通常就是"对齐两个分布"——监督学习对齐标签分布，蒸馏对齐 teacher/student，VAE 对齐 posterior/prior。

## 可收藏"记忆口诀"（尽量不用公式）

- **第一句：** 交叉熵像"总花费"，KL 像"多花的冤枉钱"。
- **第二句：** 最大似然就是把真实数据的概率抬高；取负号就变成 NLL 去最小化。
- **第三句：** one-hot 分类里，交叉熵只盯"正确类概率"，所以它看起来就是 NLL。
- **第四句：** label smoothing / 蒸馏 / VAE 的共同点：标签或隐变量不再是硬的，目标分布变"软"，所以更自然写成分布对齐。

## 收藏级对照表

| 概念 | 数学定义（尽量简洁） | 在训练里扮演的角色 | 典型场景 |
|------|-------------------|------------------|---------| 
| KL divergence（KL 散度） | $KL(p\|\|q)=E_p[\log(p/q)]$ | "分布差距"度量；最小化 KL 等价于对齐 p 和 q | 监督学习中的"对齐真实分布"、VAE 中"对齐先验" |
| Cross Entropy（交叉熵） | $H(p,q)=E_p[-\log q]$ | 常见可优化 loss；p 固定时等价于最小化 KL | 监督分类、语言模型训练（token NLL） |
| NLL（negative log-likelihood） | $NLL(\theta)=-\sum \log q_\theta(data)$ | 把"最大似然"变成"最小化"的 loss 形式 | 逻辑回归、softmax 分类、概率回归 |
| Log-likelihood（对数似然） | $\ell(\theta)=\sum \log q_\theta(data)$ | 直接的 MLE 目标（最大化） | 统计建模、生成模型密度估计 |
| MLE（maximum likelihood estimation） | $\hat{\theta}=\arg\max_\theta \ell(\theta)$ | 参数估计原则；在深度学习中常对应最小化 NLL/CE | 从线性模型到神经网络的"默认训练法" |

**表中关键等价关系与工程实现要点：** `CrossEntropyLoss` 吃 logits、等价 `LogSoftmax+NLLLoss`、label smoothing 改变目标分布等，参考课程讲义与官方文档。
